# Panda Pencil UI Brief

## 目标
使用 Pencil MCP 重新梳理 Panda 的移动端优先界面语言，先产出可以指导前端实现的高保真界面方向。

## 这次喂给 Pencil 的核心要求

### 总体风格
- 白底，尽量纯白，不要偏灰、偏黄、偏奶油。
- 整体极简、安静、轻量，减少按钮感和装饰感。
- 用暖橙和黄橙做点缀，但只能作为重点强调，不能大面积铺色。
- 卡片和浮层依赖很轻的边框和阴影建立层级，不做厚重拟物。
- 移动端优先，页面要像原生 App，而不是压缩版桌面网页。

### 会话页
- 系统一进入就回到上次会话，不需要单独首页。
- 顶部只保留左侧侧边栏唤起按钮和当前会话标题，不显示无关信息。
- 侧边栏唤起按钮必须有背景，而且风格更克制、更极简。
- 消息区参考 Remodex：
  - 系统 / assistant 在左侧，尽量是文档流，不要重聊天气泡。
  - 用户消息在右侧，允许用很轻的浅色气泡。
  - 只保留内容本身，不显示头像、角色名、额外标签。
- 输入区是带圆角矩形的浮层，白色、轻阴影、轻边框。
- 模型和推理强度按钮放在右侧，和提交按钮平行排列。
- 模型与推理强度按钮不显示“模型”“推理强度”文字标签，不要背景胶囊，只显示当前值。
- 提交按钮更小、更紧凑。
- 思考中时，提交按钮变成中断按钮，不再单独放一个中断按钮。
- 模型和推理强度弹层顶部标题分别为“选择模型”“选择推理强度”。

### 侧边栏
- 左侧侧边栏为滑出式，不是永久占位式。
- 侧边栏展开时，右侧主会话界面要一起向右滑动。
- 在会话界面用手指右滑也能唤出侧边栏。
- 侧边栏必须是纯白，不要灰底或明显脏色。
- 侧边栏顶部功能区顺序为“节点列表”“新线程”，左侧配极简图标。
- 功能区去掉强按钮样式，改成简洁列表项。
- 下方展示当前节点的线程列表。
- “工作室主楼”等命名全部统一改为“线程”。
- 每个线程下面显示该线程所有会话。
- 会话列表不要背景块，尽量是简洁列表，仅当前项做很轻的高亮。
- 会话时间放右侧，显示为“xx分钟”“xx小时”“xx天”。

### 新线程 / 新会话
- 点击“新线程”后，右侧区域切换为项目目录选择页。
- 目录树由 agent 提供懒加载接口。
- 选择目录后：
  - 如果线程已存在，就在线程下新增会话。
  - 如果线程不存在，就先建线程再建会话。

### 节点列表页
- 白底页面。
- 每个节点用白色圆角卡片承载。
- 卡片通过极轻边框和阴影与背景区分。
- 展示节点名称和 IP。
- 绿色圆点表示在线，灰色圆点表示离线。

## 这次产出的关键画面
- 会话页默认态
- 侧边栏展开态
- 节点列表页
